# Design principle 

1. Each service owns writes to its domain tables.
2. Other services call APIs or consume events.

# Transactions 

- Transaction Rule for Your Microliths

     Each service should follow this principle:
     A service owns its transaction boundary. Other services are updated through events or async workflows.

     Example: 
     	Vehicle Service creates vehicle
	Vehicle Service commits vehicle to database
	Vehicle Service publishes VehicleCreated event
	Search Service consumes event
	Search Service updates search index

CREATE TABLE outbox_event (
    event_id UUID PRIMARY KEY,
    aggregate_type VARCHAR(100),
    aggregate_id UUID,
    event_type VARCHAR(100),
    payload JSONB NOT NULL,
    status VARCHAR(20) DEFAULT 'PENDING',
    created_at TIMESTAMPTZ DEFAULT now(),
    published_at TIMESTAMPTZ,
    retry_count INT DEFAULT 0
);
